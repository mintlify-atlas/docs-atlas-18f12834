description: "Create your first custom block in CraftEngine with a texture, recipe, and loot table — from config file to in-game in under 10 minutes."

This guide takes you from a freshly installed CraftEngine to a working custom block in-game. You will configure the resource pack host so players can receive the pack, write a block configuration file based on the built-in `amethyst_torch` example, reload the plugin, and verify everything works with in-game commands.

## Prerequisites

- CraftEngine is installed and the server has started at least once (see [Installation](/installation)).
- You have operator permissions or a permissions plugin granting `ce.command.admin.*`.

## Step 1 — Set up the resource pack host

<Steps>
  <Step title="Open config.yml">
    Open `plugins/CraftEngine/config.yml` and find the `resource-pack.delivery.hosting` section:

    ```yaml
    resource-pack:
      delivery:
        send-on-join: true
        kick-if-declined: true
        kick-if-failed-to-apply: false
        prompt: "<yellow>To fully experience our server,<newline>please accept our custom resource pack.</yellow>"
        auto-upload: true
        resend-on-upload: true
        file-to-upload: "./generated/resource_pack.zip"
        hosting:
          - type: "self"
            ip: "localhost"
            port: 8163
            protocol: "http"
            deny-non-minecraft-request: true
            one-time-token: true
            strict-validation: false
            rate-limiting:
              max-bandwidth-per-second: 5_000_000
              min-download-speed-per-player: 50_000
              qps-per-ip: 5/60
    ```
  </Step>

  <Step title="Set the correct IP">
    The default value for `ip` is `localhost`, which only works when you are connecting from the same machine as the server. For any external players, replace `localhost` with your server's public IP address or domain name:

    ```yaml
    hosting:
      - type: "self"
        ip: "your.server.ip"   # replace with your actual IP or hostname
        port: 8163
        protocol: "http"
    ```

    <Tip>
    For local testing (single-player or LAN), `localhost` is fine and requires no change.
    </Tip>

    <Warning>
    Make sure port `8163` (or whichever port you configure) is open in your firewall and forwarded if you are behind NAT.
    </Warning>
  </Step>

  <Step title="Verify delivery settings">
    The three most important delivery flags are:

    | Key | Default | Effect |
    |-----|---------|--------|
    | `send-on-join` | `true` | Sends the resource pack to players automatically when they join |
    | `kick-if-declined` | `true` | Kicks players who decline the resource pack prompt |
    | `auto-upload` | `true` | Uploads the generated ZIP to the configured host automatically after generation |

    Keep these at their defaults while getting started.
  </Step>
</Steps>

## Step 2 — Create your first custom block

CraftEngine organises content under `plugins/CraftEngine/resources/<pack-name>/configuration/`. The built-in `default` pack ships with `amethyst_torch.yml` as a complete reference. You can add your own pack or place files directly in `default/configuration/blocks/`.

<Steps>
  <Step title="Create a block configuration file">
    Create a new file at:

    ```
    plugins/CraftEngine/resources/default/configuration/blocks/my_glowing_block.yml
    ```

    Below is a minimal block definition modelled after the `amethyst_torch` example. It registers one item and one block that emits light and drops itself:

    ```yaml
    items:
      default:my_glowing_block:
        data:
          item-name: "<!i>My Glowing Block"
        texture: minecraft:block/custom/my_glowing_block

    blocks:
      default:my_glowing_block:
        loot:
          template: default:loot_table/basic
          arguments:
            item: default:my_glowing_block
        settings:
          template:
            - default:sound/wood
            - default:hardness/none
          overrides:
            push-reaction: destroy
            replaceable: false
            luminance: 15
            item: default:my_glowing_block
        state:
          state: redstone_torch[lit=false]

    recipes:
      default:my_glowing_block:
        type: shaped
        pattern:
          - 'A'
          - 'B'
        ingredients:
          A: minecraft:glowstone_dust
          B: minecraft:stick
        result:
          id: default:my_glowing_block
          count: 1
    ```

    <Note>
    The `texture` field under `items` points to a resource pack path. Place your texture PNG at `plugins/CraftEngine/resources/default/resourcepack/assets/minecraft/textures/block/custom/my_glowing_block.png` before reloading so the resource pack can include it.
    </Note>
  </Step>

  <Step title="Understand the amethyst_torch reference">
    The full `amethyst_torch.yml` from the default pack shows how a more complex block works — including wall-mounted variants, directional state properties, and particle effects. Key sections to notice:

    ```yaml
    # Item section — inventory item, model, and placement behaviour
    items:
      default:amethyst_torch:
        data:
          item-name: <!i><lang:item.default.amethyst_torch>
        texture: minecraft:block/custom/amethyst_torch
        behavior:
          - type: wall_block_item
            block: default:amethyst_wall_torch
          - type: block_item
            block: default:amethyst_torch

    # Block section — server-side settings, visual state, and particles
    blocks:
      default:amethyst_torch:
        loot:
          template: default:loot_table/basic
          arguments:
            item: default:amethyst_torch
        settings:
          template:
            - default:sound/wood
            - default:hardness/none
          overrides:
            push-reaction: destroy
            replaceable: false
            map-color: 24
            luminance: 15
            item: default:amethyst_torch
        state:
          state: redstone_torch[lit=false]
          entity-renderer:
            item: default:amethyst_standing_torch
            scale: 1.01
        behavior:
          - type: sturdy_base_block
            direction: down
            support-types:
              - center
          - type: liquid_flowable_block
          - type: simple_particle_block
            tick-interval: 10
            particles:
              - particle: smoke
                x: 0.5
                y: 0.7
                z: 0.5
              - particle: dust
                color: 138,43,226
                x: 0.5
                y: 0.7
                z: 0.5

    # Recipe section — shaped crafting recipe
    recipes:
      default:amethyst_torch:
        type: shaped
        pattern:
          - 'A'
          - 'B'
        ingredients:
          A: minecraft:amethyst_shard
          B: minecraft:stick
        result:
          id: default:amethyst_torch
          count: 1
    ```

    Every item and block ID follows the pattern `<namespace>:<id>`, where `namespace` matches the `namespace` field in `pack.yml` — `default` for the built-in pack.
  </Step>
</Steps>

## Step 3 — Reload CraftEngine

After saving your YAML file, reload the plugin without restarting the server:

```bash
/ce reload
```

CraftEngine re-reads all configuration files, regenerates the resource pack ZIP, and re-uploads it to the configured host. If `resend-on-upload` is `true` (the default), connected players receive the updated pack automatically.

<Warning>
If your YAML contains syntax errors, CraftEngine logs them to the console and skips the affected file. Check the console output after running `/ce reload` to catch any issues before testing in-game.
</Warning>

## Step 4 — Test in game

Once the reload completes and you have accepted the resource pack prompt:

<Steps>
  <Step title="Get the item">
    Use the `item get` command to put the new item in your hand:

    ```bash
    /ce item get default:my_glowing_block
    ```

    You should see the item appear in your inventory with the name you configured.
  </Step>

  <Step title="Place the block">
    Place the item in the world. If you set `luminance: 15`, the block will emit full light. If you configured a recipe, you can also craft it from the ingredients you defined.
  </Step>

  <Step title="Browse all registered items">
    Run `/ce` (with no arguments) to open the item browser GUI and confirm your new block appears in the list alongside the default content.
  </Step>
</Steps>
